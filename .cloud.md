# MemeDaily State Snapshot

## Current Goal
- Make the installed PWA's top system area and sticky header remain the same opaque `#fafafa`
  surface in iOS Chrome dark mode, without moving content beneath the status bar.

## Current Architecture
- Next.js static export serves two git-backed feeds at `https://memedaily.fun`:
  memes in `data/daily/` and everyday-life news in `data/daily-news/`.
- GitHub Actions runs the primary cloud publishers at about 06:00 (news) and 07:00
  (memes), with catch-up, fallback, monitor, CI, and Pages workflows.
- The model-facing jobs may write only the exact dated JSON artifact and cannot run
  shell commands. A separate trusted job stamps publication time, validates, commits,
  and pushes. See ADR-006.
- Local Codex publishing is a supervised recovery path, not a mechanically isolated
  unattended publisher. Its rules live in `ai/prompts/CODEX_FULL_HANDOFF.md`.
- Collection is public-web intelligence only. The project stores compact source
  metadata, never login cookies, copied media, or comment archives.

## Active Work
- The root document now opts out of automatic darkening with `color-scheme: only light` and
  emits matching light/dark `theme-color` metadata.
- Critical inline CSS paints `html` and `body` before hashed assets load. The full stylesheet
  keeps `html`, `body`, `.shell`, and the sticky topbar opaque and removes backdrop blur.
- Manifest launch/background colors, offline fallback, and service-worker generation are
  aligned to `#fafafa`; a focused static regression suite protects the contract.
- Full validation, lint, typecheck, 83 tests, and the 124-page static build pass. Emitted HTML,
  manifest, service worker, inline CSS, and dark/light mobile browser assertions all satisfy
  the PWA surface contract; production deployment verification remains in progress.

## Persistent Risks
- Final status-area rendering is controlled partly by the installed iOS/WebKit container and
  requires one physical-device confirmation after Chrome refreshes the app manifest/cache.
- A previously shared Aliyun AccessKey was re-enabled outside the repository. Rotate or
  revoke it in Alibaba Cloud; secret scanning cannot invalidate an already exposed key.
- Public-source semantics require editorial judgment. Deterministic gates deliberately
  cover high-signal terms and cannot prove every archived item is contextually safe.
- GitHub Pages and this public repository are not private storage. Never commit secrets,
  private company data, login state, or unpublished source material.
